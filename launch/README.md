# Pick1 — Launch Copy Pack

Ready-to-ship copy for the four highest-leverage launch surfaces. Each file
is final-form — drop it in, tweak the bits in `[brackets]` if you want, and
publish.

| File | Where to use it | When |
|---|---|---|
| `producthunt.md` | Product Hunt launch page | Day of public launch (Tue/Wed best) |
| `hacker-news.md` | "Show HN" submission | Same day as PH or +1d, US morning |
| `linkedin-twitter.md` | Personal LinkedIn post + Twitter thread | Same day as above |
| `newsletter-pitch.md` | Cold pitch to sports/AI newsletters | 1 week before launch |

## Order of operations on launch day

1. **08:00 ET** — Push the LinkedIn post (warmest audience first)
2. **08:30 ET** — Drop the Twitter thread (links to the LinkedIn post for
   social proof)
3. **09:00 ET** — Submit Hacker News "Show HN" (US morning is the optimal
   window — peak HN traffic)
4. **09:00 ET sharp** — Product Hunt goes live the moment the new day clicks
   over PT (i.e., 00:01 PT). Schedule the launch for that window. The PH
   community votes hardest in the first 4 hours.
5. **Throughout the day** — reply to every HN/PH comment within 30 min.
   Engagement velocity is the ranking signal on both platforms.
6. **+1d** — Send the newsletter pitches you prepped a week earlier.

## What I haven't drafted (and why)

- Press release: only worth doing if you have a real press contact.
  Otherwise it sits in a void.
- TikTok / Instagram Reels: format-dependent on whether you're on camera.
  Better to script those when we know the visual approach.
- Reddit posts: rules vary wildly by subreddit and most ban product
  promotion. If you have a reputation in r/sportsbook or r/algobetting,
  worth a tailored "I built this" post — but those need to feel native
  to the community, can't be templated.
